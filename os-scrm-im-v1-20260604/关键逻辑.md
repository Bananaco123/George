# OS-SCRM IM 原型关键逻辑

> 本文档覆盖记录当前最新原型逻辑，用于后续 Demo 反推 PRD。当前仍为 `v1` 原地修改，不新建 `v2`。

## 0. 基本信息

| 项 | 内容 |
|---|---|
| 项目 | OS-SCRM · WhatsApp SCRM IM |
| 当前原型 | `pm_agent/projects/OS-SCRM/04-原型demo/v1/` |
| 真实页面 | `/scrm` |
| 本轮修改 | 在 V1 内直接调整，不新增版本 |

## 1. 原型范围

V1 聚焦 WhatsApp 会话复刻与 Web IM 工作台：

- 多 WhatsApp 账号侧边栏切换
- 当前账号工作区
- 会话列表、标签组筛选、向下加载更多
- 单聊 / 群聊混排
- 会话与聊天区之间拖拽调整宽度
- 消息流、消息 hover 操作条、失败重发
- 顶部头像 / 更多菜单打开联系人或群管理抽屉
- 群资料、成员管理、管理员调整、入群审批
- 当前账号掉线遮罩与重新扫码登录
- 最右侧 CRM / ERP 功能预留区

## 2. 布局逻辑

1. 页面主体结构为：账号栏 / 会话列表 / 可拖拽分隔线 / 聊天区 / CRM ERP 预留区。
2. 会话列表区和聊天区之间的分隔线可拖拽，用户可调整两区占比。
3. 最右侧区域不再承载 WhatsApp 群资料或联系人资料，固定预留给 SCRM 原生 CRM 关系和 ERP 交互能力。
4. 群资料、联系人信息、群成员管理通过聊天顶部头像或更多菜单打开宽抽屉承载。

## 3. 多账号逻辑

1. 员工可同时绑定多个 WhatsApp 账号。
2. 左侧账号栏展示账号头像、当前选中态、未读数、红点和闪烁提醒。
3. 当前账号工作区只展示当前账号下的会话和群聊。
4. 非当前账号收到新消息时，只更新该账号头像红点、未读数和闪烁，不自动切换当前账号。
5. 当前账号掉线时，只遮罩该账号工作区，其他在线账号仍可切换。
6. 账号栏顶部提供 `+` 新增账号入口，打开云端登录二维码弹窗。
7. 鼠标移入账号头像时，账号区域向右平滑展开，展示账号名、手机号和退出账号入口。
8. 账号 hover 展开仅向右侧溢出；退出账号必须二次确认，确认后从当前 Demo 中移除该账号及其会话。
6. 掉线恢复路径为重新扫码登录。

## 4. 会话逻辑

1. 当前账号下单聊和群聊默认混排，按置顶优先和最后活跃时间排序。
2. 会话顶部只保留搜索框和 `+` 新聊天入口，不展示当前账号姓名、手机号和在线状态。
3. 标签组按账号独立，包含所有、未读、特别关注、群组、DIY 等当前账号已有标签组。
4. 会话列表和每个标签筛选结果支持向下滑动加载更多。
5. 打开会话后，当前会话未读数清零并移除未读标签。
6. 标签组末尾固定展示 `+`，可创建自定义列表并通过二级弹窗添加当前账号下的用户或群组；创建后列表真实加入标签组，所选会话同步加入该列表。
7. 聊天顶部不单独展示标记未读和置顶按钮，相关操作收纳到三点菜单。
8. 三点菜单按信息查看、消息操作、会话状态、分组管理、发起群聊的逻辑顺序排列，支持搜索聊天内容、选择消息、更改分组、标记未读、置顶、静音、归档等当前 Demo 能力。
9. 接口未纳入或未实现的功能不展示，也不使用置灰按钮占位。
10. 会话列表卡片的三点菜单不展示搜索聊天和选择消息；会话卡片不展示静音铃铛标识。
11. 聊天顶部三点菜单中的搜索聊天打开独立搜索侧栏；搜索结果提供一键定位，点击后关闭侧栏、滚动到目标消息并短暂闪动。

## 5. 消息逻辑

1. 消息类型覆盖文本、图片、语音、文件、位置、系统消息等展示形态。
2. 消息 hover 后展示独立操作条，不依赖右键，不把“三个点”挤在气泡内部。
3. 快捷操作包含引用回复、群内 @ 成员、表情回应和更多菜单。
4. 更多菜单包含置顶、转发、编辑、撤回等消息操作。
5. 发送失败消息展示失败原因，并提供重发入口。
6. 点击重发后，原消息状态从失败变为已送达，不新建重复消息。
7. 群事件进入消息时间线，用灰色系统消息样式展示。
8. 引用回复不触发 Toast，而是在输入框上方展示引用卡片；发送后消息气泡内展示被引用内容。
9. 表情回应不触发 Toast，而是在当前消息下方展示单个表情回应，重复点击会覆盖为最新表情。
10. 输入框采用单行复合输入条：左侧为附件入口和表情入口，中间输入文本，右侧为语音/发送入口，不另起工具栏。
11. 附件入口包含文档、相册、音频、联系人卡片；前三项调起本地文件选择，联系人卡片打开选择弹窗并可发送名片消息。
12. 点击语音入口后切换为录音条，包含删除、录音时长、波形、暂停和发送入口。
13. 三点菜单中的“选择消息”仅负责进入多选模式；退出多选通过底部操作栏的关闭按钮完成。
14. 多选模式为可选消息展示复选框，底部操作栏展示已选数量，并支持将所选消息标为星标和转发。
15. 转发弹窗支持搜索并勾选目标会话；确认后目标会话真实新增消息，消息气泡展示“已转发”标识。
16. 发出的消息展示可点击的已读、已送达等状态；点击后打开信息详情侧栏。
17. 单聊信息详情展示对方的已读时间或送达时间；群聊信息详情按成员拆分已读用户和已送达用户，并展示尚未已读人数。
18. 消息状态图标统一为：灰色转圈表示待发送、单灰勾表示已发送、双灰勾表示已送达、双蓝勾表示已读、红色感叹号表示发送失败。
19. 发出消息提供“双方撤回”和“仅我们删除”；双方撤回受 2 分钟时限限制，成功后消息实时变为“此消息已撤回”，超时失败需明确提示原因。
20. 聊天内语音消息支持点击播放动效；图片、文件和位置附件支持点击打开预览弹窗。
21. 已发出的文本消息支持编辑；提交后原消息内容直接更新，并在消息时间旁展示“已编辑”。
22. 群聊消息的 `@` 快捷操作直接将该消息发送者插入输入框，不触发 Toast，也不展示 `@` 按钮的 MouseOver 提示。

## 6. 群聊逻辑

1. 群聊是核心场景，不作为单聊附属能力。
2. 本期只做手动建群，不做自动建群。
3. 手动建群入口包含会话列表“新聊天 > 新建群组”、联系人选择建群、单聊发起群聊、群详情继续加人。
3.1 “新聊天”同时提供“添加联系人”，支持通过手机号或 WhatsApp 账号搜索并生成会话。
4. 群资料和成员管理在宽抽屉中承载，包含群资料编辑、添加成员、成员管理、入群审批、退出群聊。
5. 群资料顶部卡片保留四个字段：群头像、群名、群备注、创建时间，四项均可编辑。
6. 群头像点击后打开二级弹窗上传头像；群名、群备注、创建时间通过字段后的编辑按钮进入编辑态。
7. 添加成员不在一级抽屉内直接完成，而是打开二级弹窗；弹窗支持按姓名和手机号模糊搜索，输入后展示联想人员并可直接添加。
8. 添加成员弹窗同时提供邀请链接入群方式，链接为网页地址，支持复制链接和重置链接。
9. 群成员列表中管理员永远置顶。管理员仅展示身份标签，不展示成员操作菜单。
10. 非管理员成员行展示三点菜单，支持设为群组管理员、踢出群组。
11. 当前账号自身如果是管理员，不允许对自己执行设为管理员或踢出操作。
12. 群成员操作会写入系统消息，进入群消息时间线。
13. 一个群组仅允许一个管理员；将成员设为管理员时，原管理员自动降为普通成员。
14. 非管理员成员不展示任何身份标签；创建时间只读，不提供编辑入口。
15. 入群申请模块不展示数字角标。
16. 群聊顶部仅展示群组和成员数量入口；成员在线状态、地区和当地时间直接展示在群聊信息抽屉的成员列表中。
17. 群聊抽屉标题为“群聊管理”；所有非管理员成员均展示三点操作入口，入群申请同意按钮使用蓝底白字。

## 7. 视觉与体验

1. 会话列表和聊天区采用接近 WhatsApp 的灰棕、灰黄色背景和浅绿色发送气泡，提升熟悉感。
2. 右侧保留 OS/SCRM 的业务预留表达，不与 WhatsApp 操作混在窄侧栏内。
3. 单聊顶部只展示地区和完整当地时间，不展示客户身份、在线状态或手机号；MouseOver 分别提示“地区/时间”和“当地时间”。
4. 单聊头像展示当前 WhatsApp 账号标识，MouseOver 可查看账号名称与手机号。
4. 消息区域禁止横向滚动，附件和语音卡片需在聊天区内自适应。

## 8. Demo 数据

V1 使用 localStorage 持久化 mock 状态，包括：

- 当前账号
- 账号掉线状态
- 会话未读状态
- 标签筛选结果
- 失败消息重发状态
- 群成员和入群审批状态
- 新建群聊结果

页面左侧账号栏底部提供重置按钮，可恢复初始演示数据。
